## GUI v3

Запуск:

```bash
python -m app.gui.main
```

Темы разделены:

```text
app/gui/themes/light.py
app/gui/themes/dark.py
```

Добавление камеры через GUI создаёт папки:

```text
input_stream/live/<camera_name>/
input_stream/archive/<camera_name>/
```

Новая структура видео:

```text
input_stream/archive/perimeter_camera_1/test_video.mp4
input_stream/live/perimeter_camera_1/chunk_001.mp4
```

Пайплайн ищет видео рекурсивно внутри папки:

```bash
python -m app.main --source input_stream/archive --config configs/camera_1.json
python -m app.main --source input_stream/live --config configs/camera_1.json --watch
```
## GUI v4

Добавлено:
- заголовок "Главное меню" кликабельный и возвращает на стартовый экран;
- камеры периметра должны начинаться с `per`;
- камеры территории должны начинаться с `ter`;
- запрещено создавать камеры с одинаковым названием;
- камера создаёт папки:
  - `input_stream/live/<camera_name>/`
  - `input_stream/archive/<camera_name>/`
  - `configs/<camera_name>.json`
- вкладка "Камеры периметра" показывает карточки камер;
- карточка камеры показывает preview из первого видео;
- если видео нет, показывается `NO VIDEO`;
- кнопки: удалить камеру, добавить/редактировать линии;
- интерактивная разметка:
  - 2 точки забора;
  - 4 точки стерильной зоны;
  - сохранение в `configs/<camera_name>.json`.

Запуск GUI:

```bash
python -m app.gui.main
```

Запуск обработки:

```bash
python -m app.main --source input_stream/archive
python -m app.main --source input_stream/live --watch
```

Если видео лежит в:

```text
input_stream/archive/per_cam_1/video.mp4
```

то будет использован конфиг:

```text
configs/per_cam_1.json
```

## GUI v5

Добавлено:
- камеры и настройки сохраняются между перезапусками через:
  - `input_stream/live/<camera>/`
  - `input_stream/archive/<camera>/`
  - `configs/<camera>.json`
- live-обработка стартует только по кнопке внизу;
- при запуске приложения обработка на паузе;
- при событии:
  - `WARNING_STERILE_ZONE` → карточка камеры и сводки становится жёлтой;
  - `ALARM_*` → карточка камеры и сводки становится красной;
- в кнопках "Камеры периметра" и "Сводка" появляется красный индикатор;
- в preview камеры подставляется кадр события;
- сводка читает JSON-логи и показывает интерпретацию по каждой камере.

Запуск:

```bash
python -m app.gui.main
```

## GUI v8 / ReID

- Добавлен отдельный контур мультикамерного трекинга через torchreid + OSNet.
- Камеры трекинга создаются с префиксом `trk`.
- Детекция забора и ReID работают параллельными контурами.
- В сводке добавлен фильтр: все камеры, камеры детекции, камеры трекинга.
- Архив сохраняет события обоих контуров.
- `Ок` сбрасывает цвет только для конкретной камеры и конкретного контура.

## GUI v9

- Убрана отдельная вкладка камер трекинга.
- Tracking-камеры теперь отображаются во вкладке `Камеры территории`.
- `Камеры периметра` показывают только detection-камеры с типом perimeter.
- `Камеры территории` показывают territory detection-камеры и tracking-камеры.
- У tracking-камер нет кнопки разметки линий: они работают по всей области кадра.

## GUI v10 preview logic

- Если видео обработано без событий, preview камеры обновляется каждым 15-м кадром.
- Если найдено событие, preview становится первым кадром события.
- Если были и warning, и alarm, preview остаётся/становится кадром alarm.
- Если live-обработка не запущена, но в папку камеры добавлено видео, GUI берёт первый или последний кадр и ставит его как preview без обработки.

## GUI v13 — ReID without torchreid

- Полностью убран `torchreid`.
- Больше не нужен `deep-person-reid`, `gdown`, C++ Build Tools.
- ReID работает через `torch + torchvision`.
- По умолчанию используется `mobilenet_v3_small` как embedding extractor.
- Можно поставить `resnet18` в `configs/<camera>.json`.
- Интерфейс сохранён: позже можно заменить внутренности на настоящий OSNet `.pth`.

## GUI v15 shared ReID

Исправлено:
- ReID global ID теперь общий для всех tracking-камер во время live-обработки.
- ID больше не перезапускается отдельно для каждой камеры/видео.
- На видео теперь пишется `GID` и `LID`.
- `LID` — локальный ByteTrack ID внутри камеры.
- `GID` — общий ID между tracking-камерами.
- Для MobileNetV3-Large threshold по умолчанию снижен до `0.62`.

Важно: нужно заново прогнать видео. Старые annotated-video не изменятся.
