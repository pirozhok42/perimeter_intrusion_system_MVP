# Система обнаружения нарушения периметра v0.18.2

## Основные возможности

### Детекция и тревоги
- Обнаружение людей и транспортных средств с помощью YOLO11s
- Отслеживание объектов ByteTrack
- Настраиваемая линия ограждения
- Настраиваемая стерильная зона
- Предупреждение при входе объекта в стерильную зону
- Тревога при касании или пересечении линии ограждения
- Настраиваемое время реакции
- Сохранение кадров событий
- Автоматическое создание превью камер

### Межкамерный трекинг
- Поддержка нескольких камер территории
- Глобальные идентификаторы объектов (Global ID)
- Сохранение идентификатора объекта между камерами
- ReID на основе MobileNetV3-Large
- Возможность дальнейшей замены модели ReID без изменения архитектуры

### Логирование и архив
- Логи событий в форматах JSON и CSV
- Архив событий по каждой камере
- История обнаружений и тревог
- Аннотированные выходные видеозаписи
- Сохранение конфигурации камер и интерфейса между перезапусками

### Интерфейс оператора
- Светлая и темная темы
- Авторизация пользователя
- Управление камерами периметра
- Управление камерами территории
- Интерактивная настройка линии ограждения и стерильной зоны
- Просмотр сводки событий
- Просмотр архива событий
- Поддержка обработки нескольких камер

### Режимы обработки
- Один поток
- Два потока (детекция + трекинг) — режим по умолчанию
- Поток на каждую камеру

## Структура моделей

В корне проекта должны находиться:

```text

├── mobilenet_v3_large.pth
```

## Структура конфигурации камеры

Конфигурация каждой камеры хранится в:

```text
configs/<camera_name>.json
```

Настраиваемые параметры:

- веса модели;
- пороги детекции;
- время реакции;
- размеры стерильной зоны;
- линия ограждения;
- параметры ReID;
- директории сохранения видео и логов.

Изменение параметров возможно без изменения исходного кода.
